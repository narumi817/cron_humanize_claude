# RailsレビューSkill

現在のRails実装をレビューしてください。

以下の観点を重視してください:

- Railsの慣習に沿っているか
- Controllerが肥大化していないか
- Service Objectの使い方が適切か
- 命名に一貫性があるか
- 過剰設計になっていないか
- 不要な抽象化がないか
- テストカバレッジは十分か
- MVPとして適切なスコープか
- コードから明らかなコメントは避ける
- 複雑なロジックのみ説明コメントを追加する
- 保守性向上に役立つ場合のみYARDコメントを使う

レビュー形式:

1. 良い点
2. 問題点
3. 修正提案
4. 必須修正（Must fix）と、余裕があれば対応する項目（Nice to have）の分類

厳しめに、ただし実践的にレビューしてください。
開発者は経験のあるRailsエンジニアである前提でレビューしてください。
保守性とシンプルさを重視してください。
