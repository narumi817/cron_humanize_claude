# MVP Site Quality Review Skill

MVP Webサービスの公開後品質をレビューしてください。

目的:
- 最低限の公開品質を満たす
- SEO最低限を整える
- 初見ユーザーに安心感を与える
- 将来的なAdsense審査に備える
- 過剰実装を避ける

前提:
- 個人開発
- 小さいMVP
- 低コスト運用
- 30分単位で改善

観点:

## Site credibility

- 運営者情報
- お問い合わせ
- 信頼感
- 初見ユーザー向け説明

## SEO minimum

- title
- meta description
- OGP
- sitemap
- robots
- canonical

## Operations pages

必要なページを整理:
- プライバシーポリシー
- 運営者情報
- 利用規約
- FAQ
- お問い合わせ

## Monetization readiness

将来的なAdsense観点:
- 不足ページ
- 信頼性
- コンテンツ不足

出力形式:

1. must（今やる）
2. should（必要になったら）
3. later（後回し可）
4. 個人MVPとして不要な作業

各項目について:
- なぜ必要か
- 最低限の実装内容
- MVP向けの実装方針

MVP前提なので過剰実装を避けてください。

30分以内で対応できる改善を優先してください。
